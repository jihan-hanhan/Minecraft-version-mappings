# Minecraft version mappings

**这个仓库用来记录自Minecraft 26.1以后的新版本所对应的旧版本号**  

其编排方式分为两种:  

## 1.按照年份编排

~~这可能是最容易到2.0的编排方式~~

按照Minecraft官方的说法,现在的更新频率为每年多次的小更新 ~~基本就是季节更新~~  

>Java版正式发布时将版本号重置为Java版1.0.0，此时的版本格式为1.<主要版本号>.<次要版本号>  ......  2026年起，Java版的版本格式调整为由年份、小更新发布序号及热修复发布序号组成，格式为YY.D.H，其中YY为两位年份，D为当年正式更新序号，H为热修复版本   ---引用自Minecraft Wiki

那么按照这个规律,从1.21.x为起点出发,每年为一个主要版本号,每个当年正式序号为一个次要版本号,热修复与预览版暂时忽略不计.


## 2.按照更新内容大小编排

由于Minecraft更新内容变得细碎,因此可以累计更新内容到达一定量或重要程度时,新启一个主要版本号,否则仅叠加次要版本号  ~~这会包含编者的一些主观意愿~~
