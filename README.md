# Minecraft Version Mappings

**This repository maps the new Minecraft version numbers (post-26.1 naming reform) to their legacy equivalents.**

There are two approaches to organizing these mappings:

## 1. Year-based Mapping

~~This is probably the fastest way to reach version 2.0.~~

According to Mojang, the current update frequency involves multiple minor updates per year ~~essentially seasonal updates~~.

> The full release of Java Edition ... reset the version number to 1.0.0. ... Starting in 2026, release versions are formatted as "YY.D.H", where "YY" is the two-digit year, "D" is the drop number within that year, and "H" is the hotfix counter for that drop. ... --- Quoted from the Minecraft Wiki

Following this pattern, starting from 1.21.x, each year represents a major version number, and each official drop within the year represents a minor version number. Hotfixes and snapshots/pre-releases are temporarily ignored.

## 2. Content-based Mapping

Since Minecraft updates have become more fragmented, a new major version number can be assigned when the accumulated update content reaches a certain volume or level of importance; otherwise, only the minor version number is incremented. ~~This approach involves some subjective judgment from the editors.~~
